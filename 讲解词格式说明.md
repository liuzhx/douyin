# 景点讲解词数据格式说明

## 目录结构

```
douyin-live-ai-assistant/data/scripts/
├── spot-01-故宫.json
├── spot-02-长城.json
├── spot-03-颐和园.json
├── spot-04-天坛.json
└── ...
```

## JSON 文件格式

每个景点讲解词文件包含以下字段：

```json
{
  "id": "景点唯一标识符",
  "name": "景点名称",
  "category": "景点分类（历史文化/自然风光/现代建筑等）",
  "scripts": [
    {
      "title": "讲解标题",
      "content": "讲解文本内容（建议 25-35 秒语音长度）",
      "duration_seconds": 30
    }
  ],
  "qa_knowledge": {
    "关键词1": "答案内容",
    "关键词2": "答案内容"
  }
}
```

## 讲解词编写建议

### 内容长度
- 每段讲解词建议 **25-35秒** 的语音长度
- 约 **60-100字** 中文文本
- 过长会导致观众失去耐心，过短则内容不够充实

### 内容结构
1. **开场吸引注意**："大家好！现在为大家介绍..."
2. **核心信息介绍**：景点历史、特色、亮点
3. **实用信息**：门票、开放时间、交通
4. **游览建议**：最佳路线、拍照点、体验项目

### 语言风格
- **口语化**：模拟真人导游的讲解方式
- **生动有趣**：加入小故事、历史典故
- **简洁明了**：避免专业术语，通俗易懂
- **热情友好**：营造亲切的直播氛围

## 使用方式

1. 将景点讲解词 JSON 文件放入 `data/scripts/` 目录
2. 系统启动时自动加载所有 JSON 文件
3. 支持热更新：添加新文件后无需重启服务
4. 讲解顺序：
   - **顺序模式**：按文件名字母顺序播放
   - **随机模式**：随机选择景点和讲解词

## Q&A 知识库

`qa_knowledge` 字段为该景点的常见问题提供快速答案：
- 当观众提问包含关键词时，优先使用预设答案
- 如无匹配，则调用 AI 大模型生成回答
- 提高回复准确性和响应速度
